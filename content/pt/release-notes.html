• Suporte a múltiplas instâncias: o Beagle, antes da versão 2.1, se baseava em singletons e, por isso, não podia ser instanciado mais de uma vez. Esse problema foi resolvido e mesmo que o Beagle continue funcionando como antes (não há quebras de contrato), agora também é possível usar um modo de inicialização que possibilita múltiplas configurações em um único projeto (útil para aplicações multi-módulo). [iOS | Android].

  • O ListView e o GridView agora criam o contexto implícito "index" que conta a posição do item atual na lista. Para mudar o nome desse contexto, a propriedade "indexName" foi criada.
  • Quando um item do ListView ou GridView é alterado de dentro da Lista ou Grid, essa mudança agora propaga para todos os componentes que dependem do mesmo data source.
  • As operações eq, gt, lt, gte, lte, sum, subtract, multiply e divide agora fazem coerção implícita de tipo, i.e. eles operam com tipos de dados diferentes entre si. Exemplos: eq('18', 18) retorna true, mesmo que os tipos sejam diferentes; sum(10, 20.2) retorna 30.2, mesmo que o primeiro argumento seja int e o segundo double.
  • Novas operações: int, double e string. Elas aceitam um único argumento e convertem o tipo de dado.
  • Corrige o problema one um espaço podia causar erro de leitura de uma operação dentro de uma expressão.
  • O Image Provider agora suporta aplicações multi-módulo.
  • O BeagleScreenViewController agora é público.
  • Corrige problema relacionado ao imageMode em um componente de imagem no React.
